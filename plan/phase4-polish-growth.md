### Day 69（周四）— 无障碍基础 🔨

#### 🧠 独立思考（25 分钟）

**为什么要学无障碍？**
- 无障碍是专业前端开发者的标志
- 很多公司（尤其海外）把无障碍作为硬性要求
- 好的无障碍设计对所有人都有帮助（比如键盘导航对效率工具用户很重要）

**自己检查 bookmark**：
1. **键盘导航**：只用 Tab 和 Enter，能完成核心操作吗？
   - 能导航到所有按钮和链接吗？
   - 焦点顺序合理吗？（从上到下，从左到右）
   - 有焦点样式吗？（用户能看到当前焦点在哪）

2. **语义化 HTML**：
   - 标题层级对不对？（h1 → h2 → h3，不要跳级）
   - 表单有 label 吗？
   - 按钮的文本是否描述了操作？（"删除"比"X"好）

3. **颜色对比度**：
   - 用 Chrome DevTools 检查文字和背景的对比度
   - WCAG AA 标准：普通文字 4.5:1，大文字 3:1

- 让 AI 添加 ARIA 属性（aria-label, aria-describedby 等）

### Day 70（周五）— 原理日：TS 类型系统思维 📖

#### 🧠 独立思考（全天，60-90 分钟）

**核心观点**：TypeScript 不只是"给 JS 加类型标注"

**带着这些问题去学**：

1. **类型是文档**（15 分钟）：
   - 好的类型定义让代码自解释
   - 比如看到 `function getBook(id: string): Promise<Book | null>`，不看实现就知道它做什么
   - 审查 bookmark 的函数签名，类型是否足够表达意图？

2. **泛型的本质**（20 分钟）：
   - 泛型 = 参数化类型 = 让代码既灵活又安全
   - `function first<T>(arr: T[]): T | undefined` → 输入什么类型的数组，返回什么类型
   - bookmark 中有没有地方可以用泛型简化？

3. **判别联合**（20 分钟）：
   - bookmark 的书籍有三种格式：EPUB / PDF / Web
   - 用判别联合表达：
   ```typescript
   type EPUBBook = { format: 'epub'; cfi: string; chapters: Chapter[] }
   type PDFBook = { format: 'pdf'; page: number; totalPages: number }
   type WebBook = { format: 'web'; url: string; scrollPosition: number }
   type Book = EPUBBook | PDFBook | WebBook
   ```
   - 这比 `format: string` + 一堆可选字段好在哪？

4. **unknown vs any**（10 分钟）：
   - `any` 关闭了类型检查，`unknown` 要求你先验证类型才能使用
   - 审查 bookmark 中所有 `any`，哪些可以替换为更精确的类型？

#### 🤖 AI 使用方式

- 你自己学完后，让 AI 帮你检查理解是否正确
