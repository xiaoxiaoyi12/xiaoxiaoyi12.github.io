### Day 3（周三）— NestJS 项目搭建 + 用户注册/登录

> ⚠️ **时间提示**：今天内容较多。建议 Day 2 完成数据库设计后，提前用 AI 跑通 `nest new bookmark-server` + Prisma 集成（约 15 分钟），这样今天可以专注在认证逻辑上。如果时间仍然不够，可以把"请求校验（class-validator）"推到 Day 4 一起做。

#### 🧠 独立思考（30 分钟）

**拆解训练**：今天的任务是"实现用户注册和登录"。先不写代码，拆解：

> 注册流程：
>
> 1. 前端发送 email + password → 2. 后端验证格式 → 3. 检查 email 是否已存在
>    → 4. 密码用 bcrypt 加密 → 5. 存入数据库 → 6. 返回成功
>
> 登录流程：
>
> 1. 前端发送 email + password → 2. 后端查找用户 → 3. bcrypt 比对密码
>    → 4. 密码正确 → 签发 JWT → 5. 返回 token
>
> 需要的模块：
>
> - AuthController（接收请求）
> - AuthService（业务逻辑）
> - UserService（数据库操作）
> - JwtStrategy（token 验证策略）

**你自己回答这些问题**（写在纸上）：

- JWT token 里应该存什么信息？（userId? email? role? 不要存密码!）
- token 过期时间设多久合理？（1 小时？1 天？1 周？各有什么利弊？）
- 密码有什么安全要求？（最少几位？要不要包含特殊字符？）

#### 🤖 AI 辅助（40 分钟）

让 AI 生成以下代码，你逐个 Review：

1. **NestJS 项目初始化**：`nest new bookmark-server`
2. **Prisma 集成**：PrismaModule + PrismaService
3. **Auth 模块**：
   - `POST /auth/register` — 注册
   - `POST /auth/login` — 登录
   - JWT 签发和验证
4. **请求校验**：用 class-validator 或 zod 校验 email 格式、密码长度

#### 👀 Review 清单（逐条检查）

- [ ] 密码是否用 bcrypt 哈希后再存？（`bcrypt.hash(password, 10)`）
- [ ] 数据库里存的是不是哈希值而不是明文？
- [ ] JWT_SECRET 是从环境变量读的，而不是硬编码在代码里？
- [ ] 注册时如果 email 已存在，返回的是 409 Conflict 而不是 500？
- [ ] 登录失败返回的是"邮箱或密码错误"，而不是"密码错误"？（不要暴露哪个错了）
- [ ] token 过期时间是否合理？

#### 验证标准

- 用 Postman/curl 测试：
  - 注册成功 → 返回 201
  - 重复注册 → 返回 409
