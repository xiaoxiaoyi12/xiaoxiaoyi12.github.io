# 博客管理后台使用指南

在线访问：**https://xiaoxiaoyi12.github.io/admin/**

纯浏览器端 SPA，无需安装任何软件，打开即用。

---

## 快速开始

### 1. 配置 GitHub Token

首次使用必须配置 Token，否则无法读写文章。

1. 点击顶部导航栏 **设置**
2. 在 **GitHub 连接** 区域填写：
   - **Personal Access Token**：去 [GitHub Settings > Tokens](https://github.com/settings/tokens) 生成，勾选 `repo` 权限
   - **仓库**：默认已填 `xiaoxiaoyi12/xiaoxiaoyi12.github.io`，一般不用改
   - **分支**：默认 `master`
3. 点击 **测试连接**，看到绿色 `✅ 用户名` 表示成功
4. 点击 **保存设置**

> Token 存储在浏览器 localStorage 中，不会上传到任何服务器。换浏览器或清除数据后需要重新配置。

### 2. 写第一篇文章

1. 点击顶部 **新建**（或在文章列表页点 **+ 新建文章**）
2. 填写表单：
   - **类型**：日志 / 笔记 / 阅读 / 感想
   - **标题**：文章标题
   - **日期**：默认今天
   - **标签**：输入后按回车添加，可添加多个
   - **Slug**：英文短横线格式，用于生成文件名（如 `my-first-post`）
3. 在左侧编辑区写 Markdown，右侧实时预览
4. 点击 **🚀 发布** 即可推送到 GitHub

## 三个页面

### 文章列表（文章）

管理所有文章，支持四种内容类型切换：

| 标签 | 对应目录 | 说明 |
|------|----------|------|
| 日志 | `_posts/` | 每日博客日志 |
| 笔记 | `_notes/` | 学习笔记 |
| 阅读 | `_readings/` | 阅读笔记 |
| 感想 | `_thoughts/` | 生活感想 |

**可执行操作：**
- **搜索**：输入关键词搜索标题或标签
- **刷新**：重新从 GitHub 拉取文件列表
- **编辑**：点击文章标题或 ✏️ 按钮
- **删除**：点击 🗑️ 按钮（会弹出确认框）
- **加密 / 解密**：见下方加密章节

### Markdown 编辑器（新建）

左右分栏布局：左侧 Markdown 编辑，右侧实时预览。

**快捷键：**
| 快捷键 | 功能 |
|--------|------|
| `Ctrl/Cmd + S` | 保存草稿 |
| `Ctrl/Cmd + Enter` | 发布文章 |

**草稿自动保存：** 每 30 秒自动保存到浏览器 localStorage。下次打开编辑器时会提示是否恢复草稿。

**三个发布按钮：**
| 按钮 | 说明 |
|------|------|
| 💾 草稿 | 手动保存草稿到浏览器（不推送到 GitHub） |
| 🔒 加密发布 | 输入密码后加密内容再推送 |
| 🚀 发布 | 直接推送明文到 GitHub |

### 设置

| 区域 | 说明 |
|------|------|
| **GitHub 连接** | 配置 Token、仓库、分支，可测试连接 |
| **本地服务** | 配置本地 Node 服务地址（默认 `http://localhost:3001`） |
| **存储模式** | GitHub API / 本地服务 / 自动检测 |
| **外观** | 深色 / 浅色 / 跟随系统 |
| **文章模板** | 为四种文章类型设置默认模板（JSON 格式） |

修改后记得点 **保存设置**。

## 文章加密 / 解密

支持 AES-256 加密，加密后文章在博客上需要输入密码才能阅读。

### 加密已有文章

1. 在文章列表找到目标文章
2. 点击 **🔒 加密** 按钮
3. 输入密码和可选的密码提示
4. 点击 **确认加密**

### 解密已加密文章

1. 找到已加密文章（显示 🔒 已加密）
2. 点击 **🔓 解密** 按钮
3. 输入密码
4. 点击 **确认解密**

### 新建加密文章

1. 在编辑器写好内容
2. 点击 **🔒 加密发布**（而非普通发布）
3. 输入密码
4. 文章会加密后推送到 GitHub

> 浏览器端加密与 CLI 脚本（`npm run encrypt`）和博客前端（`protected-post.html`）完全兼容。

## 存储模式

导航栏右上角的徽章显示当前模式：

| 徽章 | 模式 | 说明 |
|------|------|------|
| 🟢 GitHub | GitHub API | 通过 GitHub API 直接读写仓库文件 |
| 🔵 Local | 本地服务 | 通过 `npm run admin` 启动的本地 Node 服务 |
| ⚪ Offline | 离线 | 无法连接任何后端 |

**推荐使用 GitHub API 模式**（默认），无需启动本地服务。

如需使用本地模式：
```bash
npm run admin
```
启动后在设置中将存储模式切换为「本地服务」或「自动检测」。

## 文章模板

在设置页可以为每种文章类型配置默认模板，新建文章时自动填充。

模板格式为 JSON：
```json
{
  "title": "日志 - {date}",
  "date": "{date}",
  "tags": [],
  "body": ""
}
```

- `{date}` 会自动替换为当天日期（YYYY-MM-DD）
- `body` 是 Markdown 正文的默认内容
- 点击「恢复默认」可重置为内置模板

## 常见问题

**Q: 发布文章后博客没有更新？**
A: GitHub Pages 部署需要 1-2 分钟，稍等后刷新博客页面。

**Q: 提示「请先在设置中配置 GitHub Token」？**
A: 进入设置页配置 Token 并保存。

**Q: Token 安全吗？**
A: Token 只存在浏览器 localStorage 中，所有 API 请求直接从你的浏览器发往 GitHub，不经过任何中间服务器。但建议使用最小权限的 Token（只勾选 `repo`），定期轮换。

**Q: 换了浏览器 / 电脑怎么办？**
A: 重新配置 Token 即可，文章数据都在 GitHub 仓库里。

**Q: 文章列表加载很慢？**
A: 首次加载需要逐个获取文件内容以解析标题和标签。加载完成后会缓存，切换标签页时速度会快很多。
